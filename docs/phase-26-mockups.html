Locked decisions

  • Header band: 36dp rune + 54sp level numeral + class/title meta. Right-meta column max 120dp + ellipsis on overflow.
  • Active-state glow removed. Only radiant state keeps a sweep arc.
  • Each stat row: 48dp min-height, 6dp dot, 20sp Rajdhani 700 tabular rank num, 4dp body-part-hue bar, "X XP · Y para o próximo rank" label.
  • All 6 body parts visible (cardio omitted entirely).
  • Stat rows tappable → routes to /saga/stats with body part pre-selected.
  • Dot pulse on rank-up — 24h glow-ring on the body-part's dot after a rank threshold crosses.
  • Character XP bar: 6dp height, violet gradient, label "X XP · Y para LVL N".
  • Untrained body parts render at 0.4 opacity with "—" rank.

Locked decisions

  • Trend chart: selected line 2.5dp/100%, others 1dp/35%. 180ms linear tween on tap. Lines colored by body-part identity (not vitality state). Section-header padding fixes overlap.
  • Vitality table: percentages in HP-drain ramp (high green / mid amber / low red). No "Path mastered" or "caminho esfriando" copy on active/fading rows. Only untested keeps "sem dados".
  • ⓘ icons on both vitality section headers → same bottom sheet (definition · 3-state ramp · rank-safety guarantee in heroGold).
  • Volume & pico: per-body-part two-column block. Left = Volume (X/Y séries) with weekly delta. Right = Carga pico (kg/lbs) with 30D delta badge.
  • Delta adapts to history: weeks 2–4 → "vs semana passada"; weeks 5+ → "vs média (4 sem)". Under-target red, over-target amber (not green), goal-met green bullet ●.
  • Generic-tip fallback: right column becomes "REFERÊNCIA" + 10 séries + "ⓘ estimado" → bottom-sheet explainer (Schoenfeld 2019 maintenance floor).
  • Peak Loads horizontal-bar section deleted. Peak data lives in V&P right column.

Vitality ⓘ bottom sheet

Opens from the ⓘ on either vitality section header (trend or table). Same content from both entry points. Three parts: definition, three-state ramp, rank-safety guarantee in heroGold-bordered box.

bottom sheet
Vitalidade

Vitalidade reflete o quão recente é seu treino para cada grupo muscular. É um indicador de quanto sua jornada está ativa, não da sua força.

Como ela se move:

Ativo 66–100%
Esmorecendo 34–65%
Dormente 0–33%

Sobe a cada treino do grupo. Desce gradualmente nos dias sem treino — quanto mais tempo, menor fica.

Seu rank nunca cai. Vitalidade só sinaliza atividade recente — o progresso conquistado no rank de cada grupo fica permanente.

Locked decisions

  • State band labels in pt-BR: Ativo / Esmorecendo / Dormente. Single-word RPG-flavored triad.
  • Rank-safety guarantee boxed in heroGold-bordered callout — load-bearing copy to prevent users reading red vitality as "I lost my rank."
  • Same bottom sheet entry from both vitality section headers (trend + table) — one source of truth.
  • L10n fix in 26a: vitalityCopyDormant currently says "Aguarda seu primeiro passo" (which describes Untested) — repurpose for the actual Dormant state.

Locked decisions

  • Schema: existing BucketRoutine + one field — isSpontaneous: bool (default false). No nullable routineId, no length constraint.
  • Auto-absorb on save_workout: match uncompleted bucket entry by routineId → set completedWorkoutId. No match → create new entry with isSpontaneous = true, order appended.
  • Status icons: ○ violet outline ring (planned) · ✓ green filled (planned + done) · ✓ violet filled + ★ tag (spontaneous + done).
  • Counter: unique completion days (two workouts same day = 1 day).
  • Multi-workout per day naturally supported — second workout just creates another entry.
  • Week rollover: copy entries where isSpontaneous == false; clear completion state. Spontaneous entries don't carry forward.
  • Engajamento section: 6 body-part bars in canonical order (cardio hidden). Done-fill + 40%-opacity planned-fill stacked on same track. ⓘ → set-counting rule explainer. No total counter (compound double-counting would mislead).
  • Set-counting rule (locked): primary by max XP share, ties at max count for all tied body parts. Single source: exercises.xp_attribution.

Locked decisions

  • Rotating encouragement nudge priority: cross-build → body-part title → weekly plan → streak → first-step.
  • ActionHero adaptive: "Iniciar {next}" / "Iniciar treino livre" / "Criar primeira rotina".
