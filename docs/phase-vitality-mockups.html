§2 · Conditioning charged beat — three variants for multi-BP sessions

A session can charge several body parts (legs + core + a cardio finisher). The honest per-BP recompute returns a delta for each trained group. How to present them? Three options, isolated below at full panel width for legibility. All share the teal eyebrow + the rebuild-only fill grammar.

A · Single aggregate bar

⚡ Condicionamento recarregado
Esta sessão · 3 grupos
▲+14%
A runa recarrega ao longo de 7 dias.
Recommended
One charge bar, aggregate delta. The lighter teal stub is the was snapshot; solid teal is the now post-save value. The count-up animates the solid fill from the was tick rightward only.

Pros: single calm beat, never competes with the rank-delta rows directly above it. Per-BP identity is already carried by those rows + the segmented bar, so re-coloring per BP here would be redundant noise. One number = one <2s glance. Scales to 1 BP or 6 BPs with no layout change.
Cons: hides which BP charged most — acceptable, the rank rows + bar already answer which; this beat answers did my conditioning rebuild. Aggregate = the volume-weighted EWMA roll-up across trained groups (matches the server roll-up).

B · Per-BP mini-bars

⚡ Condicionamento recarregado
Pernas▲+18%
Core▲+9%
Condicionamento▲+15%
Alternative
One mini charge bar per trained BP, hue-coded, each with its own delta, via BodyPartHues.hueFor per row.

Pros: honest granularity — you see exactly which rune charged most. Strong on cardio days where Conditioning (teal) is the star and you want it spotlighted next to strength groups.
Cons: directly duplicates the per-BP rank-delta rows AND the segmented bar already on screen — three stacked per-BP visualizations read as repetitive chrome (anti-generic-AI flag: stacked identical bars is exactly the card-soup we avoid). On a 6-BP session it bloats the scroll. Conditioning (teal) is itself one of the BPs, so it sits inline here, not separately.

C · Dominant BP only

⚡ Condicionamento recarregado
Pernas▲+18%
+ Core e Condicionamento também recarregaram.
Alternative
Spotlight the dominant BP only, tinted box in that BP hue, with a one-line "+ others also charged" footer.

Pros: picks a hero, mirrors the existing dominant-BP logic the next-target callout already uses (state.dominantBodyPart) — consistent mental model. Compact.
Cons: on a pure-cardio finisher day where Conditioning is the WHOLE point, dominant is whatever earned most volume — could bury the cardio charge under a strength group. The tinted box re-introduces a per-BP color competing with the rank-delta row directly above for the same BP. Picking a hero implies the others lost, clashing with the rebuild-not-deplete framing.

Recommendation — Variant A (single aggregate charge bar)

  • One calm beat, one number. Which-BP is already answered three lines up by the segmented bar + per-BP rank-delta rows. The conditioning beat answers a different question — did my conditioning rebuild — so it should be one honest aggregate signal, not a fourth stacked per-BP visualization.
  • Rebuild, never deplete. The bar has a dim was stub and a solid now fill; the count-up only ever moves the fill rightward from was to now. No drain, no red, no shrinking. The "▲ +14%" + the ⚡ rune + "recarrega ao longo de 7 dias" footer lock the rebuild metaphor in copy, not just motion.
  • Teal is the Conditioning hue, used once. Eyebrow + bar fill = bodyPartCardio #22D3EE. Even when the session is all-strength the aggregate bar stays teal, because the beat IS the Conditioning concept — teal is its identity, not a per-BP color here. Keeps brand violet / heroGold off the bar (scarcity contract intact).
  • Anti-generic-AI: no gradient, no rounded drop-shadow card, no decorative icon beyond the single functional ⚡ rune. The bar is the same hard geometry family as XpSegmentedBar (8px vs 16px) — a deliberate slimmer sibling so it reads as a secondary state signal under the primary XP bar, not a clone competing for weight.

Open decisions for the user

  • Scope label: "Esta sessão · 3 grupos" vs just "Condicionamento". The 3-grupos caption tells the user the aggregate spans 3 trained BPs. Drop it if it reads as clutter.
  • Show the absolute now-%? Current design shows only the delta (+14%) — honest and motivating. We could also surface the absolute charge level (72%) at the bar right tick. Risk: an absolute % invites why-isnt-it-100% anxiety that fights the rebuild framing. Recommend delta-only.
  • Single-BP sessions: with one trained group the aggregate bar and a single mini-bar are visually identical, so Variant A degrades gracefully — the N-grupos caption reads 1 grupo or is omitted.
