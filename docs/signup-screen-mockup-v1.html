Signup Screen — UX Proposals v1

Four-column comparative: today as-is (left) + three redesign directions + one lead recommendation. All proposals honor LGPD age-gate (PR #309), violet-stack/heroGold identity, Rajdhani/Barlow typography, and pt-BR primary market. Design-critique framework applied.

1. Design Critique (Current State)

Applying the design-critique framework: first impression, usability, visual hierarchy, consistency, accessibility.

First Impression (2 seconds)

The screen opens on: 48dp sigil, Rajdhani REPSAGA in hot-violet, two text fields, a button. The signup context is communicated only by the 16sp dim subtitle “Crie sua conta” and the button label “Cadastrar.” The form reads as a login screen with a confusing extra checkbox. A Brazilian user coming from Hevy or Strong will notice immediately that confirm-password is missing.

What Works Well

  • Sigil + Rajdhani REPSAGA in hot-violet: correct brand signal at top. Immediately distinctive vs generic fitness apps.
  • GradientButton for primary CTA: thumb-friendly full-width layout, correct visual weight.
  • AutofillGroup + AutofillHints.newPassword: best-practice integration already wired correctly.
  • Error box pattern (error-tinted container + liveRegion semantics): solid accessibility foundation.
  • LGPD structural guarantee (null onPressed as gate, not a runtime flag): correct architecture, just needs visual polish.

Lead Recommendation: Option A — Full-Form Signup

Pick Option A. It removes all three critical gaps (confirm-password, display name, strength indicator) in a single screen with no routing changes. The inline checkbox label solves legal-link duplication in one line of copy. The CRIAR CONTA heading upgrade costs one line of code and eliminates the mode-ambiguity issue. On 412dp (most Brazilian mid-market phones) five fields fit without hard scroll; on 360dp the user scrolls approximately one thumb-width — acceptable for a once-per-lifetime action. Implementation complexity is the lowest of the three options.

Option B is the right follow-up if post-launch analytics show Google conversion above 60% of new signups. But it sacrifices the name-field improvement and makes Google branding the dominant visual — a brand cost not worth taking before measuring. Option C is architecturally sound but adds routing complexity and a second drop-off event. Ship Option A, measure, then iterate toward B or C if the data supports it.

Brazilian-market copy notes: “Como quer ser chamado?” is warmer than “Nome de usuario.” Strength bar labels: “Adicione numeros ou simbolos” (Medium), “Otima senha!” (Strong). The inline checkbox copy should be reviewed by a native speaker before release — LGPD Art. 14 wording is a legal document concern beyond UX copy.

3. Implementation Notes for Tech-Lead (Option A)

Design notes only — not implementation specs. These document the UX intent so tech-lead can make the right technical choices.
  • Heading: Add a Rajdhani 700 16sp Text(CRIAR CONTA) widget below the app-name line, visible only when _isSignUp. Replace the dim 16sp subtitle with this heading in signup mode.
  • Name field: New AppTextField above email in signup mode. autofillHints: [AutofillHints.name]. Pass displayName to signUpWithEmail alongside locale in user_metadata. Validate non-empty only (no length minimum).
  • Confirm-password field: New AppTextField, obscureText: true, autofillHints: [AutofillHints.newPassword], signup mode only. Validator: value == _passwordController.text, error string from l10n.passwordMismatch ARB key.
  • Strength bar: Pure local computation. Score: 1 = length ≥ 6 (weak), 2 = length ≥ 8 OR contains digit OR special char (medium), 3 = all three (strong). Three Container segments colored error/warning/success. Non-blocking.
  • Inline checkbox label: Replace CheckboxListTile.title Text with Text.rich containing WidgetSpan TextButton nodes for legal links inline. Use MaterialTapTargetSize.padded (48dp floor) — the current chip row uses shrinkWrap, a BUG-018 recurrence. Remove the separate legal-chips Padding + Row entirely.
  • Footer suppression: Wrap _LegalFooter render with if (!_isSignUp). In signup mode the inline checkbox satisfies the LGPD disclosure requirement.
  • Helper text on disabled CTA: When _isSignUp && !_ageConfirmed, render Barlow 11sp text-dim Text below the CTA: l10n.signupAgeRequiredHint (Confirme sua idade para continuar).
