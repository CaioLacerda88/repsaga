# Bestiary — Full Catalog (v1 content)

The content behind `docs/bestiary-spec.md`. This is the source for the eventual
`assets/bestiary/bestiary.json`. Every slot has **2+ variants** so consecutive
similar sessions surface different beasts (anti-repetition). Resolution is
deterministic per session (see §Selection). All names en / pt.

**Hue rule (locked):** the beast name + rank sigil render in the **dominant body
part's hue**; a chimera renders a **multi-hue gradient** across the trained parts.

## 2. Bosses — PR / rank-up (named elite)

A boss = the dominant line's creature **promoted one tier**, prefixed with a gold
**epithet** (epithet chosen by session hash → variety). Format: `[Epithet], the [Creature]`
/ `[Creature], [Epítome]`.

**Epithet pool (en / pt):** Ironheart / Coração-de-Ferro · the Unbroken / o Inquebrável ·
the Sovereign / o Soberano · Dawnbreaker / Quebra-Aurora · the Undying / o Imortal ·
Stormcrown / Coroa-Tempestade · the Relentless / o Implacável · the First / o Primeiro.

> e.g. PR on a Rank-C arms day → promote to B (Chimera-Claw) → **"Ironheart, the Chimera-Claw" / "Garra-Quimera, o Coração-de-Ferro"**, boss styling (crown + gold frame + "CHEFE").

## 3. Chimeras — combinations (the part you liked)

A session that spreads XP across **2+ body parts significantly** fuses into a Chimera.
This is **generative + curated** so every combination is covered without a 21-row table,
and variety comes from each line carrying **2 fusion-adjectives**.

### 3b. Two-part rule (2 dominant lines)
`The [adj of 2nd line] [noun of dominant line]` / `O [Noun do dominante] [adj da 2ª]`.
The dominant line gives the body, the secondary gives the quality; the adjective rotates by hash.
> dominant Chest + secondary Arms → **"The Clawed Golem" / "O Golem Garrudo"** (or "Razor Golem"/"Golem Navalha").
> dominant Back + secondary Legs → **"The Earthen Drake" / "O Dragão Telúrico"**.
> dominant Arms + secondary Cardio → **"The Storm-Borne Manticore" / "A Mantícora Tempestuosa"**.

## 5. Selection (deterministic, anti-repetition)

1. **Line** = dominant body part (most session XP). **Tier** = session-XP band (spec §3).
2. **Kind:** PR/rank-up → boss (§2) · 2 dominant parts → chimera 2-part (§3b) · 3-4 → §3c ·
   5+ → §3d · milestone hit → legendary (§4, highest priority) · else → base (§1).
3. **Variant pick** = `hash(session_id) mod (variant count)` — deterministic for a given
   session, but the next similar session hashes differently → a different variant. Same for
   the boss epithet and the chimera adjective. **Never the same beast two sessions running**
   for the same (line, tier) — keep a 1-deep "last beast" guard client-side to skip a repeat.
4. **Hue:** name + sigil in `BodyPartHues.hueFor(dominant)`; chimera = gradient over the
   trained parts' hues (the part the user liked).
5. **Achievement phrase:** spec §6 (highest-priority trait).

**Catalog size v1:** 84 base + 8 epithets + 7 fusion lines (generative, covers all pairs)
+ 7 curated hybrids ×2 + 2 (3-4 part) ×2 + 3 full-body + 6 legendaries ≈ a deep, non-repetitive set.

## 6. Why this drives "do more" (retention)

Variety is the hook: training the **same split** repeatedly cycles variants (never stale),
but training **new parts / harder / balanced** unlocks **new lines, higher tiers, chimeras,
and bosses** the user hasn't seen — the collectible itch. The future **in-app Bestiary log**
(spec §9) makes the gaps explicit ("Cardio S-rank: never felled", "3 / 10 chimeras") — a
pull toward variety and intensity, framed as a record of deeds, never a chore list.
